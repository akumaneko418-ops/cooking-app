# 管理ツールの高度化（フェーズ4）実装計画

管理者向けのレシピ更新作業をより効率的かつ直感的にするため、管理ツールの機能を強化します。

## 概要
1.  **入力補助の強化**: 材料や手順の動的な追加・削除をよりスムーズに。
2.  **カテゴリ選択の高度化**: 従来の単一選択から、最新のデータ構造に合わせたマルチセレクト（複数選択）対応へ。
3.  **ビジュアルフィードバック**: 入力した画像URLのリアルタイムプレビュー機能。
4.  **バリデーションの改善**: 必須項目のチェックを強化し、保存ミスを防止。

## 変更内容

### 1. セキュアな管理者権限の保護
アプリ内から任意のユーザーがデータを書き換えられないよう、SupabaseのRLSと連携した強固なアクセス制御を導入します。
- **特権アカウント用 RLS ポリシー (`secure_admin_permissions.sql`)**: データベースのレシピ、材料、手順などのテーブルにおいて、特定の管理者（ユーザー自身が普段ログインしているメールアドレス）のみ UPDATE/INSERT/DELETE を許可します。
- **アプリ内認証処理 (`SettingsScreen.tsx` 等)**: アプリ内でログイン中のユーザーが管理者であるかを判定（メールアドレス等でチェック）し、管理者であれば編集可能にする仕様へ変更します。

### 2. [Admin] レシピ編集フォームの刷新

#### [MODIFY] [AdminRecipeFormScreen.tsx](file:///c:/Users/akuma/画像生成用/output/cooking-app/app/src/screens/AdminRecipeFormScreen.tsx)
- **カテゴリ選択**: `Picker` による単一選択から、チェックボックス形式またはタグ形式のマルチセレクトに変更。
- **画像プレビュー**: `imageUrl` 入力欄の隣に、現在のURLで表示される画像を小型プレビュー。
- **動的リスト操作**: 
    - 材料・手順の各項目に「削除」ボタンを分かりやすく配置。
    - 空の項目が残らないよう自動トリミング。
- **スタイリング**: 全体的に余白を整理し、入力しやすいプレミアムな管理者用UIへ。

## 検証プラン

### 自動テスト
- 各種入力（カテゴリ複数選択、材料追加・削除）が正しくストレージ（Supabase/AsyncStorage）に反映されるか。
- 空の画像URLや無効なURLを入力した際の挙動確認。

### 手動検証
- 実際にレシピを1件作成・編集し、ホーム画面や検索画面に正しく反映されるかを確認。
- マルチセレクトしたカテゴリが全てレシピに紐付いているか。
