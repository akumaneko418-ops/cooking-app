# 管理者用ツール（レシピ管理機能）の実装計画

ユーザー（管理者）がアプリ内のレシピを直感的に追加・編集・削除できるようにするため、データ管理を動的に行える管理者機能を実装します。

## 概要
これまでコード内に直接記述（ハードコード）されていたレシピデータを、`AsyncStorage` で管理する「マスターレシピ」として移行し、専用の管理画面から操作できるようにします。

## 変更内容

### 1. データ層とストレージの拡張
- **マスターレシピの型定義**: 一覧表示用（タイトル、時間、難易度等）と詳細表示用（材料、手順）を統合したデータ型を定義します。
- **ストレージ機能の追加**: `storage.ts` に、マスターレシピの取得・保存・削除を行う関数を追加します。
- **初期データの実装**: 初回起動時に、現在のダミーレシピを初期データとして `AsyncStorage` にロードする仕組みを構築します。

### 2. 管理画面の実装
- **高度な隠しアクセス機能**: 
  1. 設定画面の「アプリ情報（Version）」を**3秒間長押し**します。
  2. 画面上にパスコード入力用のシークレット・ダイアログが出現します。
  3. **独自の4桁パスコード（管理用）**を入力することで、初めて管理者ボタンが表示されます。
  ※これにより、一般ユーザーがいくら連打しても辿り着けない仕組みになります。
- **管理者ダッシュボード (`AdminDashboardScreen.tsx`)**: 
  - 登録されている全レシピのリスト表示、検索、削除。
  - 「新しいレシピを追加」ボタン。
- **レシピ編集フォーム (`AdminRecipeFormScreen.tsx`)**:
  - タイトル、調理時間、難易度、カテゴリ、画像URL。
  - 材料リスト（動的な追加・削除）。
  - 手順リスト（動的な追加・削除）。

### 3. 各画面の改修（動的データ対応）
- **ホーム画面 (`HomeScreen.tsx`)**: ハードコードされたデータではなく、ストレージから取得したデータを表示。
- **検索画面 (`SearchScreen.tsx`)**: 同様にストレージから取得。
- **詳細画面 (`RecipeDetailScreen.tsx`)**: ナビゲーションから渡されたIDを元に、ストレージから全詳細データを取得して表示。

### 4. ナビゲーション、設定、UI最適化
- **設定画面 (`SettingsScreen.tsx`)**: 下部にひっそりと「管理者用ツール」への入り口を追加。
- **ナビゲーター (`App.tsx`)**: 
  - 新しい管理画面をスタックナビゲーションに登録。
  - **重要**: ボトムタブバーがスマホのシステムUI（ホームインジケーター）と重ならないよう、`useSafeAreaInsets` を使用して動的に `paddingBottom` を適用します。
- **全体的なセーフエリア対応**: 各画面の最上部・最下部がノッチやインジケーターに被らないよう、`SafeAreaView` の配置を最適化します。

## 検証プラン

### 自動テスト / 手動検証
1. **レシピ追加**: 管理画面から新しいレシピを追加し、ホーム画面のリストに表示されるか確認。
2. **レシピ編集**: 既存レシピのタイトルや調理時間を変更し、詳細画面に反映されるか確認。
3. **詳細表示**: 異なるレシピIDを指定して、それぞれの材料・手順が正しく出し分けられるか確認。
4. **初期化**: ストレージをクリアしても、初期データが正しく復元されるか確認。
